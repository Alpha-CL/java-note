#### node mode

``` javascript
///////////////////////////////////////////////////////////////////////////////////////////////////////////////////////


/**
 * 节点模式: 单向链 | 双向链 | 环向链
 * 
 * 
 * 每一个几点都会包含
 * 
 *      - 上一个节点的地址
 *      - 下一个节点的地址
 *      - 当前节点的内容
 */


            第一个节点                    中间的节点                  最后一个节点
            
      + ----------------- +        + ---------------- +       + ----------------- +
      |                   |        |                  |       |                   |
      |     firstNode     | -----> |   ... Node ...   | ----> |     lastNode      |
      |                   |        |                  |       |                   |
      + ----------------- +        + ---------------  +       + ----------------- +
    
          prev: null,                 prev: firstNode             prev: Node,
          content: {}                 content: {},                content: {},
          next: Node                  next: lastNode              next: null


///////////////////////////////////////////////////////////////////////////////////////////////////////////////////////
```

## behavior design mode

#### strategy mode

``` javascript
///////////////////////////////////////////////////////////////////////////////////////////////////////////////////////


/**
 * strategy mode( 策略模式 )
 * 
 * 
 * 用于解决 执行流程固定, 但执行结果不相同
 */

                                    + ----- abstract class ---- +      // 约束不同子类相同功能方法该使用同样的使用方式
                                    |                           |
                                    |                           |
                                    |      eat();               |       * 继承: 每个子类继承后都可以使用的方法
                                    |      sleep();             |
                                    |                           |
                                    |      abstract sayHi();    |       * 多态: 子类实现不同的该方法
                                    |                           |              ( 调用该方法的方式是一样的 )
                                    |                           |
                                    + ----------- + ----------- +
                                                  |
                                                  |
                 + --------------------- + ------ + ------------ + --------------------- +
                 |                       |                       |                       |
                 |                       |                       |                       |
        + ----------------- +   + ----------------- +   + ----------------- +   + ----------------- +
        |                   |   |                   |   |                   |   |                   |
        |   child class     |   |   child class     |   |   child class     |   |      ... ...      |
        |                   |   |                   |   |                   |   |                   |
        + ----------------- +   + ----------------- +   + ----------------- +   + ----------------- +
        
        // 抽象类中的抽象方法, 子类继承后都必须实现
        ( 从而约束子类 )


///////////////////////////////////////////////////////////////////////////////////////////////////////////////////////
```
